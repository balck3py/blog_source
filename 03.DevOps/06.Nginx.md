## 一、Nginx基础定义和功能

### 1.1 什么是Nginx

Nginx（发音为"engine x"）是一个高性能的HTTP和反向代理Web服务器，同时也是一个IMAP/POP3/SMTP代理服务器。由俄罗斯程序员Igor Sysoev开发，2004年首次公开发布。

### 1.2 核心特性

- **高性能**：采用事件驱动的异步非阻塞架构，能够处理大量并发连接
- **低内存占用**：相比Apache等传统服务器，内存占用更少
- **高并发**：单机可支持数万并发连接
- **反向代理**：可作为反向代理服务器，转发请求到后端服务器
- **负载均衡**：内置多种负载均衡算法
- **静态资源服务**：高效处理静态文件请求
- **SSL/TLS终止**：支持HTTPS加密传输
- **HTTP/2支持**：支持HTTP/2协议
- **模块化设计**：丰富的模块生态系统

### 1.3 主要应用场景

1. **Web服务器**：直接提供HTTP服务
2. **反向代理**：隐藏后端服务器，提供统一入口
3. **负载均衡**：分发请求到多个后端服务器
4. **静态资源服务**：高效处理静态文件
5. **API网关**：作为微服务架构的API网关
6. **SSL终端**：处理HTTPS加密和解密
7. **内容缓存**：缓存静态和动态内容

## 二、Nginx安装与基础配置

### 2.1 安装Nginx

#### Ubuntu/Debian系统
```bash
# 更新包列表
sudo apt update

# 安装Nginx
sudo apt install -y nginx

# 启动Nginx服务
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx

# 检查状态
sudo systemctl status nginx
```

#### CentOS/RHEL系统
```bash
# 安装EPEL仓库（如果需要）
sudo yum install -y epel-release

# 安装Nginx
sudo yum install -y nginx

# 启动并设置开机自启
sudo systemctl start nginx
sudo systemctl enable nginx
```

#### 源码编译安装
```bash
# 下载源码
wget http://nginx.org/download/nginx-1.24.0.tar.gz
tar -xzf nginx-1.24.0.tar.gz
cd nginx-1.24.0

# 配置编译选项
./configure --prefix=/usr/local/nginx \
            --with-http_ssl_module \
            --with-http_realip_module \
            --with-http_stub_status_module \
            --with-http_gzip_static_module \
            --with-stream

# 编译和安装
make && sudo make install

# 启动Nginx
sudo /usr/local/nginx/sbin/nginx
```

### 2.2 基础配置文件结构

Nginx配置文件通常位于 `/etc/nginx/nginx.conf`（包管理器安装）或 `/usr/local/nginx/conf/nginx.conf`（源码安装）。

```nginx
# 主配置文件结构
# /etc/nginx/nginx.conf

# 全局配置块
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

# Events块：配置事件模型
events {
    worker_connections 1024;
    use epoll;  # Linux系统推荐使用epoll
}

# HTTP块：HTTP服务器配置
http {
    # 基础配置
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    
    access_log /var/log/nginx/access.log main;
    
    # 性能优化配置
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    
    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    
    # Server块：虚拟主机配置
    server {
        listen 80;
        server_name example.com;
        root /var/www/html;
        index index.html index.htm;
        
        location / {
            try_files $uri $uri/ =404;
        }
    }
    
    # 包含其他配置文件
    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}
```

## 三、负载均衡配置

### 3.1 负载均衡算法

Nginx支持多种负载均衡算法：

1. **轮询（Round Robin）**：默认算法，按顺序分配请求
2. **加权轮询（Weighted Round Robin）**：根据权重分配请求
3. **IP哈希（IP Hash）**：根据客户端IP分配，保证同一IP访问同一服务器
4. **最少连接（Least Connections）**：优先分配给连接数最少的服务器
5. **加权最少连接（Weighted Least Connections）**：结合权重和连接数

### 3.2 HTTP负载均衡配置

```nginx
# 定义上游服务器组
upstream backend_servers {
    # 默认使用轮询算法
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# 加权轮询
upstream backend_weighted {
    server 192.168.1.10:8080 weight=3;  # 权重3，处理更多请求
    server 192.168.1.11:8080 weight=2;  # 权重2
    server 192.168.1.12:8080 weight=1;  # 权重1
}

# IP哈希算法
upstream backend_ip_hash {
    ip_hash;  # 启用IP哈希
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# 最少连接算法
upstream backend_least_conn {
    least_conn;  # 启用最少连接
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}

# 健康检查和故障转移
upstream backend_with_health {
    server 192.168.1.10:8080 max_fails=3 fail_timeout=30s;
    server 192.168.1.11:8080 max_fails=3 fail_timeout=30s;
    server 192.168.1.12:8080 backup;  # 备用服务器
    server 192.168.1.13:8080 down;     # 标记为不可用
}

# 使用上游服务器组
server {
    listen 80;
    server_name api.example.com;
    
    location / {
        proxy_pass http://backend_servers;
        
        # 代理头设置
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
        
        # 缓冲设置
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
    }
}
```

### 3.3 高级负载均衡配置

```nginx
# 基于URI的负载均衡
upstream backend_api {
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
}

upstream backend_web {
    server 192.168.1.20:8080;
    server 192.168.1.21:8080;
}

server {
    listen 80;
    server_name example.com;
    
    # API请求转发到API服务器组
    location /api/ {
        proxy_pass http://backend_api;
    }
    
    # Web请求转发到Web服务器组
    location / {
        proxy_pass http://backend_web;
    }
}

# 会话保持（Sticky Session）
upstream backend_sticky {
    # 使用cookie实现会话保持
    sticky cookie srv_id expires=1h domain=.example.com path=/;
    server 192.168.1.10:8080;
    server 192.168.1.11:8080;
    server 192.168.1.12:8080;
}
```

## 四、多配置文件管理

### 4.1 主配置文件包含其他配置

Nginx支持通过`include`指令包含其他配置文件，实现配置的模块化管理。

```nginx
# /etc/nginx/nginx.conf (主配置文件)
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    # 基础配置
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    # 日志格式配置
    include /etc/nginx/conf.d/log-formats.conf;
    
    # 通用配置
    include /etc/nginx/conf.d/common.conf;
    
    # 上游服务器配置
    include /etc/nginx/conf.d/upstreams.conf;
    
    # 虚拟主机配置
    include /etc/nginx/sites-enabled/*.conf;
    include /etc/nginx/conf.d/*.conf;
}
```

### 4.2 分离配置文件示例

### 4.3 配置文件管理最佳实践

```bash
# 目录结构建议
/etc/nginx/
├── nginx.conf              # 主配置文件
├── mime.types              # MIME类型定义
├── conf.d/                 # 通用配置文件目录
│   ├── log-formats.conf    # 日志格式
│   ├── common.conf         # 通用HTTP配置
│   ├── upstreams.conf      # 上游服务器配置
│   └── proxy-common.conf   # 通用代理配置
├── sites-available/        # 可用的站点配置
│   ├── site1.conf
│   └── site2.conf
└── sites-enabled/          # 启用的站点配置（符号链接）
    ├── site1.conf -> ../sites-available/site1.conf
    └── site2.conf -> ../sites-available/site2.conf

# 启用/禁用站点
sudo ln -s /etc/nginx/sites-available/site1.conf /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/site1.conf

# 测试并重载配置
sudo nginx -t && sudo nginx -s reload
```

### 5.4 默认服务器和通配符配置

```nginx
# 默认服务器（处理未匹配的请求）
server {
    listen 80 default_server;
    server_name _;
    return 444;  # 关闭连接，不返回响应
}

# 通配符域名
server {
    listen 80;
    server_name *.example.com;
    root /var/www/wildcard;
}

# 精确匹配优先于通配符
server {
    listen 80;
    server_name www.example.com;
    root /var/www/www;
}
```

### 5.5 Server Name匹配规则

Nginx按以下顺序匹配server_name：

1. **精确匹配**：完全匹配的域名
2. **通配符前缀**：`*.example.com`
3. **通配符后缀**：`example.*`
4. **正则表达式**：以`~`开头
5. **默认服务器**：`default_server`或第一个匹配的server块

```nginx
# 正则表达式匹配
server {
    listen 80;
    server_name ~^www\.(.+)$;
    root /var/www/$1;  # 使用捕获组
}

# 多个server_name
server {
    listen 80;
    server_name example.com www.example.com api.example.com;
    root /var/www/example;
}
```

### 5.6 Location块配置

```nginx
server {
    listen 80;
    server_name example.com;
    
    # 精确匹配（优先级最高）
    location = / {
        return 301 /index.html;
    }
    
    # 前缀匹配（最长匹配优先）
    location /api/ {
        proxy_pass http://backend_api;
    }
    
    location /api/v1/ {
        proxy_pass http://backend_api_v1;
    }
    
    # 正则表达式匹配（按顺序匹配）
    location ~ \.(jpg|jpeg|png|gif|ico)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # 大小写敏感的正则匹配
    location ~* \.(jpg|jpeg|png|gif|ico)$ {
        expires 30d;
    }
    
    # 优先匹配（停止正则匹配）
    location ^~ /static/ {
        root /var/www;
        expires 1y;
    }
    
    # 通用匹配
    location / {
        try_files $uri $uri/ =404;
    }
}
```

## 六、自定义日志格式输出

### 6.1 日志格式变量

Nginx提供了丰富的日志变量：

```nginx
# 常用变量
$remote_addr          # 客户端IP地址
$remote_user          # 客户端用户名（HTTP认证）
$time_local           # 本地时间（格式：25/Dec/2023:10:11:23 +0800）
$time_iso8601         # ISO8601格式时间
$request              # 完整请求行
$request_method       # 请求方法（GET、POST等）
$request_uri          # 完整请求URI
$request_length       # 请求长度
$args                 # 查询参数
$query_string         # 同$args
$status               # 响应状态码
$body_bytes_sent      # 响应体大小（字节）
$bytes_sent           # 发送给客户端的字节数
$http_referer         # Referer头
$http_user_agent      # User-Agent头
$http_x_forwarded_for # X-Forwarded-For头
$request_time         # 请求处理时间（秒）
$upstream_addr        # 上游服务器地址
$upstream_status      # 上游服务器响应状态码
$upstream_response_time # 上游服务器响应时间
$upstream_connect_time   # 上游服务器连接时间
$upstream_header_time    # 上游服务器头时间
```

### 6.2 自定义日志格式示例

```nginx
# 基础日志格式
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent" "$http_x_forwarded_for"';

# 详细日志格式（包含性能指标）
log_format detailed '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for" '
                    'rt=$request_time uct="$upstream_connect_time" '
                    'uht="$upstream_header_time" urt="$upstream_response_time" '
                    'us="$upstream_status" ua="$upstream_addr"';

# JSON格式日志（便于日志分析工具解析）
log_format json escape=json '{'
    '"time":"$time_iso8601",'
    '"remote_addr":"$remote_addr",'
    '"remote_user":"$remote_user",'
    '"request_method":"$request_method",'
    '"request_uri":"$request_uri",'
    '"status":$status,'
    '"body_bytes_sent":$body_bytes_sent,'
    '"request_time":$request_time,'
    '"http_referer":"$http_referer",'
    '"http_user_agent":"$http_user_agent",'
    '"http_x_forwarded_for":"$http_x_forwarded_for",'
    '"upstream_addr":"$upstream_addr",'
    '"upstream_status":"$upstream_status",'
    '"upstream_response_time":"$upstream_response_time"'
'}';

# 组合日志格式（类似Apache）
log_format combined '$remote_addr - $remote_user [$time_local] '
                    '"$request" $status $body_bytes_sent '
                    '"$http_referer" "$http_user_agent"';

# 性能分析日志
log_format performance '$time_iso8601|$remote_addr|$request_method|$request_uri|'
                       '$status|$request_time|$upstream_response_time|'
                       '$body_bytes_sent|$http_user_agent';

# 使用自定义日志格式
server {
    listen 80;
    server_name example.com;
    
    # 使用JSON格式
    access_log /var/log/nginx/example.access.json json;
    error_log /var/log/nginx/example.error.log warn;
    
    # 不同location使用不同日志格式
    location /api/ {
        access_log /var/log/nginx/api.access.log detailed;
        proxy_pass http://backend_api;
    }
    
    location / {
        access_log /var/log/nginx/web.access.log main;
```

## 八、TCP四层和HTTP七层代理

### 8.1 TCP/UDP四层代理（Stream模块）

Nginx的Stream模块提供了TCP和UDP四层代理功能，类似于L4负载均衡器。

#### 基础TCP代理配置
```nginx
# 在nginx.conf的顶层（不在http块内）
stream {
    # 日志格式
    log_format basic '$remote_addr [$time_local] '
                     '$protocol $status $bytes_sent $bytes_received '
                     '$session_time';
    
    access_log /var/log/nginx/stream.access.log basic;
    
    # TCP代理到后端服务器
    upstream backend_tcp {
        server 192.168.1.10:3306;  # MySQL
        server 192.168.1.11:3306;
    }
    
    # TCP服务器配置
    server {
        listen 3306;
        proxy_pass backend_tcp;
        proxy_timeout 1s;
        proxy_responses 1;
        error_log /var/log/nginx/tcp_mysql.error.log;
    }
    
    # UDP代理
    upstream dns_servers {
        server 8.8.8.8:53;
        server 8.8.4.4:53;
    }
    
    server {
        listen 53 udp;
        proxy_pass dns_servers;
        proxy_timeout 1s;
        proxy_responses 1;
    }
}
```

#### TCP负载均衡
```nginx
stream {
    # TCP负载均衡（轮询）
    upstream mysql_cluster {
        server 192.168.1.10:3306 weight=2;
        server 192.168.1.11:3306 weight=1;
        server 192.168.1.12:3306 backup;
    }
    
    # TCP负载均衡（最少连接）
    upstream redis_cluster {
        least_conn;
        server 192.168.1.20:6379;
        server 192.168.1.21:6379;
        server 192.168.1.22:6379;
    }
    
    # TCP负载均衡（IP哈希）
    upstream tcp_backend {
        hash $remote_addr consistent;
        server 192.168.1.10:8080;
        server 192.168.1.11:8080;
        server 192.168.1.12:8080;
    }
    
    server {
        listen 3306;
        proxy_pass mysql_cluster;
        proxy_connect_timeout 1s;
        proxy_timeout 3s;
        proxy_buffer_size 4k;
    }
    
    server {
        listen 6379;
        proxy_pass redis_cluster;
    }
}
```

#### TCP SSL/TLS代理
```nginx
stream {
    # SSL上游服务器
    upstream ssl_backend {
        server 192.168.1.10:443;
    }
    
    # SSL终端（终止SSL）
    server {
        listen 443 ssl;
        ssl_certificate /etc/nginx/ssl/server.crt;
        ssl_certificate_key /etc/nginx/ssl/server.key;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        
        proxy_pass ssl_backend;
        proxy_ssl on;
        proxy_ssl_verify off;
    }
    
    # SSL透传（SSL转发）
    server {
        listen 8443 ssl;
        ssl_certificate /etc/nginx/ssl/server.crt;
        ssl_certificate_key /etc/nginx/ssl/server.key;
        
        # SSL转发到后端
        proxy_pass 192.168.1.10:443;
        proxy_ssl on;
        proxy_ssl_verify off;
        proxy_ssl_name backend.example.com;
    }
    
    # SSL预读（根据SNI路由）
    map $ssl_preread_server_name $backend {
        mysql.example.com backend_mysql;
        redis.example.com backend_redis;
        default backend_default;
    }
    
    upstream backend_mysql {
        server 192.168.1.10:3306;
    }
    
    upstream backend_redis {
        server 192.168.1.20:6379;
    }
    
    upstream backend_default {
        server 192.168.1.30:8080;
    }
    
    server {
        listen 443;
        proxy_pass $backend;
        ssl_preread on;  # 启用SSL预读
    }
}
```

### 8.2 HTTP七层代理

HTTP代理是Nginx的核心功能，工作在应用层（OSI第7层）。

#### 基础HTTP代理
```nginx
http {
    upstream backend {
        server 192.168.1.10:8080;
        server 192.168.1.11:8080;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        location / {
            proxy_pass http://backend;
            
            # 代理头设置
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # 超时设置
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
            
            # 缓冲设置
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
        }
    }
}
```

#### HTTP SSL/TLS终止
```nginx
http {
    server {
        listen 443 ssl http2;
        server_name example.com;
        
        # SSL证书配置
        ssl_certificate /etc/nginx/ssl/example.com.crt;
        ssl_certificate_key /etc/nginx/ssl/example.com.key;
        
        # SSL协议和加密套件
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;
        ssl_prefer_server_ciphers off;
        
        # SSL会话缓存
        ssl_session_cache shared:SSL:10m;
        ssl_session_timeout 10m;
        
        # OCSP Stapling
        ssl_stapling on;
        ssl_stapling_verify on;
        ssl_trusted_certificate /etc/nginx/ssl/ca.crt;
        
        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto https;
        }
    }
    
    # HTTP重定向到HTTPS
    server {
        listen 80;
        server_name example.com;
        return 301 https://$server_name$request_uri;
    }
}
```

#### HTTP到HTTPS代理（SSL转发）
```nginx
http {
    upstream backend_https {
        server 192.168.1.10:443;
    }
    
    server {
        listen 80;
        server_name example.com;
        
        location / {
            proxy_pass https://backend_https;
            
            # SSL验证
            proxy_ssl_verify on;
            proxy_ssl_trusted_certificate /etc/nginx/ssl/ca.crt;
            proxy_ssl_verify_depth 2;
            proxy_ssl_name backend.example.com;
```

### 8.3 四层和七层代理对比

| 特性 | TCP四层代理（Stream） | HTTP七层代理 |
|------|---------------------|------------|
| **工作层级** | OSI第4层（传输层） | OSI第7层（应用层） |
| **协议支持** | TCP、UDP | HTTP、HTTPS、HTTP/2 |
| **性能** | 更高（无应用层解析） | 相对较低（需要解析HTTP） |
| **功能** | 简单转发 | 丰富的HTTP功能（路由、重写、缓存等） |
| **SSL处理** | SSL透传或终止 | SSL终止、SSL转发 |
| **适用场景** | 数据库、Redis、自定义TCP协议 | Web应用、API服务 |
| **负载均衡** | 基于IP和端口 | 基于URL、Header等 |

## 十一、总结

掌握这些配置和最佳实践，可以构建高性能、高可用的Web服务架构。
